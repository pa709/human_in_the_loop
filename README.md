# Vision-Blackout Recovery via Human Gesture Imitation

A system that recovers robot manipulation tasks after a vision blackout by mapping human arm gestures (tracked via ArUco marker) to robot actions using learned sequence-to-sequence models.

## Installation

### 1. Python dependencies

```bash
pip install -r requirements.txt
```

> `opencv-contrib-python` is required (not plain `opencv-python`) for the `cv2.aruco` module. Do not install both — they conflict.

### 2. robomimic and robosuite

These are not on PyPI and must be installed separately:

```bash
# robosuite
pip install robosuite

# robomimic
git clone https://github.com/ARISE-Initiative/robomimic.git
cd robomimic && pip install -e .
```

## Pipeline

### Step 0 — Setup (run once)

**Generate the ChArUco calibration board** (print and use for camera calibration):

```bash
cd setup/
python checkeraruco.py
# output: charuco_board.png (save in setup/)
```

**Generate the ArUco hand marker** (print and attach to back of hand):

```bash
python generate_aruco.py
# outputs: aruco_marker_id0.png, aruco_marker_A4.png (save in setup/)
```

**Calibrate the webcam** (point at printed ChArUco board, press ENTER ~15–20× at varied angles):

```bash
python camera_calibration.py \
    --camera_index 1 \
    --output ../calibration/calibration.npz
```

---

### Step 1 — Collect robot trajectories *(Linux / GPU server)*

Requires the trained robomimic checkpoint.

```bash
cd collection/
python collect_success_robot_trajectories.py \
    --agent ../checkpoints/model_epoch_150_Lift_success_0_96.pth \
    --n_success 50 \
    --output_path ../data/raw/robot_trajectories.hdf5
```

Each demo in the output HDF5 contains the frozen RGB frame, EEF pose at blackout, and ground-truth remaining robot actions.

---

### Step 2 — Record human gestures *(Mac, with EMEET webcam)*

Copy `data/raw/robot_trajectories.hdf5` from Step 1 to the Mac first.

```bash
cd collection/
python record_human_trajectories.py \
    --hdf5 ../data/raw/robot_trajectories.hdf5 \
    --camera_name "EMEET SmartCam C960 4K" \
    --calib ../calibration/calibration.npz \
    --duration 120 \
    --output_dir /tmp/blackout_frames
```

Output is saved automatically alongside the input HDF5 as `recorded_vision_blackout_trajectories.hdf5`. Copy it back to `data/raw/` on the server.

To test the pipeline without writing any data:

```bash
python record_human_trajectories.py --hdf5 ../data/raw/robot_trajectories.hdf5 --dry_run
```

> To find your camera name: `ffmpeg -f avfoundation -list_devices true -i ""`

---

### Step 3 — Preprocess

```bash
cd preprocessing/
python preprocess_v2.py \
    ../data/raw/recorded_vision_blackout_trajectories.hdf5 \
    ../data/processed/preprocessed_human.hdf5
```

This cleans raw marker trajectories: drops startup bursts, fills small gaps (100–500 ms) via interpolation, trims end spikes, detects motion start, and temporally rescales each demo to match its robot trajectory length.

---

### Step 4 — Train

All training scripts share the same argument order: `PREPROC.hdf5 ROBOT.hdf5 OUT_DIR`.

```bash
cd training/

python train_mlp.py \
    ../data/processed/preprocessed_human.hdf5 \
    ../data/raw/robot_trajectories.hdf5 \
    ../runs/mlp/

python train_lstm.py \
    ../data/processed/preprocessed_human.hdf5 \
    ../data/raw/robot_trajectories.hdf5 \
    ../runs/lstm_ss/

python train_lstm_tf.py \
    ../data/processed/preprocessed_human.hdf5 \
    ../data/raw/robot_trajectories.hdf5 \
    ../runs/lstm_tf/
```

Each run saves a model checkpoint and `train_log.json` to the specified output directory.

---

### Step 5 — Evaluate (offline)

```bash
cd evaluation/

python evaluate_mlp.py \
    ../data/processed/preprocessed_human.hdf5 \
    ../data/raw/robot_trajectories.hdf5 \
    ../runs/mlp/model_mlp.pt

python evaluate_lstm.py \
    ../data/processed/preprocessed_human.hdf5 \
    ../data/raw/robot_trajectories.hdf5 \
    ../runs/lstm_ss/seq2seq_model.pt
```

Reports per-demo and mean L2 error on position and rotation channels, in both teacher-forcing and autoregressive modes.

---

### Step 6 — Summarize rollouts

After running rollouts in simulation and collecting a `rollout_results.json`:

```bash
cd evaluation/
python summarize_rollouts.py rollout_results.json --csv summary.csv
```

Prints a success-rate table with Wilson confidence intervals and a per-demo symbol breakdown.

## Models

| Model | Definition file | Training script | Checkpoint name |
|---|---|---|---|
| MLP baseline | `models/model_mlp.py` | `train_mlp.py` | `model_mlp.pt` |
| LSTM-SS (scheduled sampling) | `models/model_lstm.py` | `train_lstm.py` | `seq2seq_model.pt` |
| LSTM-TF (teacher forcing) | `models/model_lstm_tf.py` | `train_lstm_tf.py` | `model.pt` |

All models take a 6-channel human gesture trajectory (Δtvec + Δrvec, delta from trajectory start) as input and predict a 6-DOF robot action sequence (first 6 dims of OSC_POSE actions).

## Data Format

### `data/raw/robot_trajectories.hdf5`

```
data/
  demo_N/
    frozen_rgb            (256, 256, 3)  uint8   — last RGB frame before blackout
    eef_at_blackout       (7,)           float64 — EEF pos (3) + quat (4) at blackout
    remaining_actions     (T, 7)         float64 — ground-truth robot actions post-blackout
    remaining_eef_poses   (T, 7)         float64 — ground-truth EEF poses post-blackout
    attrs: blackout_idx, total_traj_len, remaining_len
```

### `data/raw/recorded_vision_blackout_trajectories.hdf5`

```
data/
  demo_N/
    human_tvecs           (N, 3) — raw marker translation vectors (metres)
    human_rvecs           (N, 3) — raw marker rotation vectors (radians)
    human_timestamps      (N,)   — seconds since recording start
    attrs: demo_idx, remaining_len, n_human_poses
```

### `data/processed/preprocessed_human.hdf5`

```
data/
  demo_N/
    human_tvecs_scaled    (T, 3) — cleaned and rescaled to match robot T
    human_rvecs_scaled    (T, 3)
    human_tvecs_raw       (N, 3) — original raw data preserved
    human_rvecs_raw       (N, 3)
    human_timestamps_raw  (N,)
    attrs: dropped_burst, edge_lead, edge_trail, interpolated,
           motion_start, motion_end, snippet_n
```

## Known Issues

- `checkeraruco.py` uses the older `CharucoBoard_create` API which is deprecated in OpenCV ≥ 4.6. It still works but may show a deprecation warning. The newer `camera_calibration.py` already uses the current `CharucoBoard(...)` API.
- The `--camera_name` argument in `record_human_trajectories.py` must exactly match the device string shown by: `ffmpeg -f avfoundation -list_devices true -i ""`
